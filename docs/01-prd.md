# Animal Daily — 产品需求文档 (PRD)

> 版本: 1.0  
> 创建日期: 2026-03-12  
> 状态: 草稿

---

## 1. 产品概述

**Animal Daily** 是一款个人每日事项记录应用，用户可以记录每天做了哪些事情，并为每件事打上"等级评分"，从而量化自己每天的状态。

**核心理念**: 用轻松、接地气的方式记录和回顾每一天，让"今天过得怎么样"有据可查。

**目标用户**: 希望追踪日常产出、量化自我状态的个人用户（上班族、学生、自由职业者）。

---

## 2. 等级评分系统

### 2.1 等级定义（从高到低）

| 等级 | 数值 | 含义 | 图标建议 |
|------|------|------|----------|
| 🔥 夯 | 5 | 今天爆发了，产出拉满 | 火焰 |
| ⭐ 顶级 | 4 | 高效充实，超出预期 | 星星 |
| 💪 人上人 | 3 | 正常发挥，该做的都做了 | 肌肉 |
| 😐 NPC | 2 | 浑浑噩噩，随波逐流 | 面无表情 |
| 💀 拉完了 | 1 | 废了，什么都没干 | 骷髅 |

### 2.2 评分规则

- **事项级评分**: 每条事项可独立打分
- **每日综合评分**: 
  - 自动计算：取当日所有事项等级的加权平均（按时长加权）
  - 手动覆盖：用户可手动设定当日综合等级
- **无事项日**: 默认等级为"拉完了"（可在设置中关闭）

## 3. 功能需求

### 3.1 P0 — 核心功能（MVP）

#### 3.1.1 用户认证
- 注册（邮箱 + 密码）
- 登录 / 登出
- Token 自动刷新

#### 3.1.2 事项记录
- 添加事项：内容（必填）、分类（可选）、等级（必填）、耗时（可选）
- 编辑事项
- 删除事项
- 按日期查看事项列表（时间轴形式）

#### 3.1.3 每日总结
- 查看当日综合等级（自动计算）
- 手动覆盖当日等级
- 日历视图：每天显示对应等级颜色/图标

#### 3.1.4 事项分类
- 预设分类：工作、学习、生活、娱乐、运动、其他
- 自定义分类（用户可增删）

### 3.2 P1 — 增强功能

#### 3.2.1 数据统计
- 周/月等级分布饼图
- 每日等级趋势折线图
- 事项数量趋势
- 各分类时间占比
- 连续"夯"天数统计

#### 3.2.2 目标设定
- 设定周目标（如"本周至少3天人上人以上"）
- 目标达成进度展示
- 目标到期提醒

#### 3.2.3 提醒通知
- 每日记录提醒（可设定时间，默认晚上 21:00）
- 推送文案随等级变化（"今天还没记录，要变 NPC 了"）

### 3.3 P2 — 扩展功能

#### 3.3.1 AI 辅助
- 根据事项内容自动推荐等级
- 生成每日/每周文字总结
- 智能分类建议

#### 3.3.2 数据导出
- 导出为 Markdown / CSV
- 生成分享卡片（图片）
- 周报/月报生成

#### 3.3.3 Web 端（方案 C 阶段）
- 与移动端数据同步
- 更丰富的统计图表
- 键盘快捷操作

## 4. 用户故事

### 4.1 核心场景

**场景 1: 快速记录**
> 作为用户，我在完成一件事后，打开 app，花 10 秒添加一条记录并打分，然后继续做下一件事。

**场景 2: 晚间回顾**
> 作为用户，我在睡前打开 app，查看今天做了哪些事，看到综合等级是"人上人"，感觉还不错。

**场景 3: 周末复盘**
> 作为用户，我在周末打开统计页面，发现这周有 3 天是 NPC，决定下周要更努力。

**场景 4: 日历浏览**
> 作为用户，我打开日历视图，看到这个月大部分天都是绿色（人上人以上），只有周末偶尔红色（拉完了），整体趋势不错。

### 4.2 边界场景

**场景 5: 忘记记录**
> 晚上 21:00 收到推送"今天还没记录哦"，点击进入 app 补录。

**场景 6: 补录历史**
> 昨天忘了记录，今天可以切换到昨天的日期进行补录。

**场景 7: 空白日**
> 某天完全没有记录，日历上显示"拉完了"灰色标记。

## 5. 非功能需求

| 项目 | 要求 |
|------|------|
| 响应速度 | 添加事项 < 500ms，页面切换 < 300ms |
| 离线支持 | 无网络时可本地记录，联网后自动同步 |
| 数据安全 | 密码 bcrypt 加密，API 全程 HTTPS |
| 多端同步 | 移动端数据实时同步到服务端 |
| 兼容性 | iOS 15+, Android 10+ |
